# 国麻实机开桌检查清单

这份清单用于服务器实机联机验证国标麻将流程是否可用。
建议至少准备 4 个可控座位，优先使用 4 名人类玩家；如果人数不够，可混用 Bot，但吃碰杠胡的精确场景最好由真人配合验证。

## 准备

- [ ] 服务端正常加载 MahjongPaper 与 CraftEngine。
- [ ] 资源包已生效，能看到麻将牌正反面与花牌贴图。
- [ ] 测试员具备 `/mahjong create`、`/mahjong rule`、`/mahjong state`、`/mahjong forceend` 权限。
- [ ] 记下当前测试版本、世界名、参与玩家名，便于复盘。

## 开桌与切换国麻

- [ ] 执行 `/mahjong create` 创建牌桌。
- [ ] 执行 `/mahjong list` 记录 `tableId`。
- [ ] 4 名玩家分别执行 `/mahjong join <tableId>` 入座。
- [ ] 执行 `/mahjong rule variant GB`。
- [ ] 可选再执行一次 `/mahjong state`，确认当前规则摘要已经是 GB 桌。
- [ ] 4 名玩家分别执行 `/mahjong start`，确认牌局开始。

预期结果：

- [ ] 庄家起手 14 张可打牌，闲家起手 13 张可打牌。
- [ ] 桌面中央/状态摘要显示当前是 GB 规则，而不是日麻规则。
- [ ] 不应出现立直、宝牌、里宝牌相关提示。

## 花牌与补花

- [ ] 开局后持续进行，直到任意玩家起手带花或摸到花。
- [ ] 出现花牌后，确认花牌不会留在手牌可打区，而是公开显示在该玩家副露/公开牌区域。
- [ ] 确认补花后该玩家仍保持正确的可打牌张数：
  - 庄家轮到自己操作时应有 14 张可打牌。
  - 非庄家在自己摸牌后应有 14 张可打牌。
  - 非当前操作玩家通常显示 13 张可打牌。
- [ ] 若补花摸到的仍是花牌，确认会继续补花直到拿到可打牌或流局。

预期结果：

- [ ] 花牌会正确公开显示。
- [ ] 补花不会把花牌错留在手牌里。
- [ ] 花牌应计入后续和牌番种。

## 吃碰杠动作

### 吃

- [ ] 制造一个只能由下家吃的场景。
- [ ] 下家执行 `/mahjong chii <tileA> <tileB>`。
- [ ] 确认被吃的那张弃牌从河里移除。
- [ ] 确认吃牌后由吃牌者继续出牌，且不会额外摸一张普通牌。

### 碰

- [ ] 制造一个可碰的场景。
- [ ] 执行 `/mahjong pon`。
- [ ] 确认被碰的弃牌从河里移除。
- [ ] 确认碰牌后由碰牌者继续出牌，且不会额外摸一张普通牌。

### 明杠

- [ ] 制造一个可明杠的场景。
- [ ] 执行 `/mahjong minkan`。
- [ ] 确认被杠的弃牌从河里移除。
- [ ] 确认明杠后会从牌墙尾部补牌。
- [ ] 若补牌是花牌，确认会继续补花。

### 暗杠 / 加杠

- [ ] 制造一个可暗杠的场景，执行 `/mahjong kan <tile>`。
- [ ] 确认暗杠后会从牌墙尾部补牌。
- [ ] 制造一个碰后加杠的场景，执行 `/mahjong kan <tile>`。
- [ ] 确认加杠后也会从牌墙尾部补牌。

### 抢杠和

- [ ] 制造一个加杠时其他玩家可和的场景。
- [ ] 其他玩家执行 `/mahjong ron`。
- [ ] 确认荣和优先于加杠结算。

## 和牌与 8 番门槛

- [ ] 先尝试一个明显不能和或不足 8 番的 `/mahjong tsumo` / `/mahjong ron`。
- [ ] 确认命令失败，不应错误结束对局。
- [ ] 再完成一个合法的 8 番或以上和牌。
- [ ] 自摸时执行 `/mahjong tsumo`，荣和时执行 `/mahjong ron`。

预期结果：

- [ ] 不足 8 番时不能和。
- [ ] 满足条件时能正常和牌并结算。
- [ ] 结算 UI 显示总番数与番种列表。
- [ ] 若本局含花牌，番种列表里应能反映花牌番。

## 多人反应优先级

- [ ] 制造一张弃牌同时可被多个玩家响应的场景。
- [ ] 优先验证 `ron > minkan > pon > chii`。
- [ ] 未选择动作的玩家执行 `/mahjong skip`。

预期结果：

- [ ] 多人反应按优先级正确结算。
- [ ] 未被采用的低优先级动作不会错误生效。

## 结算与下一局

- [ ] 和牌或流局后，确认 `/mahjong settlement` 能重新打开结算界面。
- [ ] 确认一局结束后玩家需要再次 `/mahjong start` 才会开始下一局。
- [ ] 执行 `/mahjong state`，确认轮次、庄位、剩余状态刷新正常。
- [ ] 连续打至少 2 局，确认不会因为切到 GB 后第二局开始失败。

## 建议记录

- [ ] 记录每个通过项对应的截图或短视频。
- [ ] 如出现异常，记录：
  - 触发命令
  - 桌号 `tableId`
  - 参与座位
  - 出问题前最后一张弃牌/摸牌/花牌
  - `/mahjong state` 输出

## 一轮最小通过标准

- [ ] 能成功从 `/mahjong rule variant GB` 开局到完整结算。
- [ ] 花牌与补花至少验证通过 1 次。
- [ ] 吃、碰、明杠、暗杠/加杠、自摸、荣和至少各成功 1 次。
- [ ] 不足 8 番不能和，满足条件可以和。
- [ ] 第二局能正常继续开始。
